# Next.js App Router 완벽 가이드

Next.js 13에서 도입되고 14에서 안정화된 App Router는 React의 최신 기능들을 활용한 새로운 라우팅 시스템입니다.

## App Router vs Pages Router

### Pages Router (레거시)

```
pages/
├── index.tsx
├── about.tsx
└── posts/
    └── [id].tsx
```

### App Router (최신)

```
app/
├── page.tsx
├── layout.tsx
└── posts/
    └── [id]/
        └── page.tsx
```

## 핵심 개념

### 1. Server Components (기본값)

App Router의 모든 컴포넌트는 기본적으로 Server Component입니다:

```typescript
// app/page.tsx
export default function HomePage() {
  // 이 컴포넌트는 서버에서만 실행됩니다
  return <h1>Home Page</h1>;
}
```

**장점:**
- 번들 크기 감소
- 서버 리소스 직접 접근
- SEO 최적화

### 2. Client Components

상호작용이 필요한 경우에만 `'use client'` 지시어를 사용합니다:

```typescript
'use client';

import { useState } from 'react';

export function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}
```

### 3. Layout과 Template

#### Layout (상태 유지)

```typescript
// app/layout.tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="ko">
      <body>{children}</body>
    </html>
  );
}
```

#### Template (상태 초기화)

```typescript
// app/template.tsx
export default function Template({
  children,
}: {
  children: React.ReactNode;
}) {
  return <div>{children}</div>;
}
```

## 데이터 페칭

### Server Component에서

```typescript
async function getPosts() {
  const res = await fetch('https://api.example.com/posts');
  return res.json();
}

export default async function PostsPage() {
  const posts = await getPosts();

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```

### 캐싱 전략

```typescript
// 기본: 무한 캐시
fetch('https://api.example.com/data');

// Revalidate: 10초마다 갱신
fetch('https://api.example.com/data', {
  next: { revalidate: 10 }
});

// No cache: 항상 최신 데이터
fetch('https://api.example.com/data', {
  cache: 'no-store'
});
```

## 라우팅 패턴

### 동적 라우트

```typescript
// app/posts/[slug]/page.tsx
export default function PostPage({
  params,
}: {
  params: { slug: string };
}) {
  return <h1>Post: {params.slug}</h1>;
}

// 정적 생성을 위한 경로 생성
export async function generateStaticParams() {
  const posts = await getPosts();

  return posts.map((post) => ({
    slug: post.slug,
  }));
}
```

### 병렬 라우트

```typescript
// app/dashboard/layout.tsx
export default function Layout({
  children,
  analytics,
  team,
}: {
  children: React.ReactNode;
  analytics: React.ReactNode;
  team: React.ReactNode;
}) {
  return (
    <>
      {children}
      {analytics}
      {team}
    </>
  );
}
```

### 인터셉팅 라우트

모달을 구현할 때 유용합니다:

```
app/
├── feed/
│   └── page.tsx
└── photo/
    ├── [id]/
    │   └── page.tsx
    └── (..)photo/
        └── [id]/
            └── page.tsx
```

## 스트리밍과 Suspense

```typescript

export default function Page() {
  return (
    <>
      <Header />

    </>

```

## 메타데이터 API

### 정적 메타데이터

```typescript
export const metadata = {
  title: 'Next.js App Router',
  description: 'Modern web development',
};
```

### 동적 메타데이터

```typescript
export async function generateMetadata({
  params,
}: {
  params: { id: string };
}) {
  const post = await getPost(params.id);

  return {
    title: post.title,
    description: post.excerpt,
  };
}
```

## 모범 사례

1. **Server Component를 기본으로 사용하세요**
   - Client Component는 필요할 때만 사용

2. **적절한 캐싱 전략을 선택하세요**
   - 정적 데이터: 기본 캐시 사용
   - 동적 데이터: revalidate 설정
   - 실시간 데이터: no-store

3. **Streaming을 활용하세요**
   - 느린 컴포넌트를 Suspense로 감싸기
   - 점진적 렌더링으로 UX 개선

4. **코드 분할을 의식하세요**
   - Dynamic import 활용
   - 라우트별 자동 코드 분할
