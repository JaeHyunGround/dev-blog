# 효율적인 Git 워크플로우 팁

협업 프로젝트에서 Git을 더 효율적으로 사용하는 방법을 알아봅니다.

## 브랜치 전략

Feature Branch 워크플로우를 추천합니다.

```bash
git checkout -b feature/new-feature
git add .
git commit -m "feat: 새로운 기능 추가"
git push origin feature/new-feature
```

## 유용한 Git 명령어

### 커밋 수정하기

```bash
git commit --amend
```

### 마지막 커밋 취소하기

```bash
git reset --soft HEAD~1
```

### 특정 파일만 스테이징

```bash
git add -p
```

## 커밋 메시지 컨벤션

```
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 포맷팅
refactor: 코드 리팩토링
test: 테스트 추가
```

깔끔한 Git 히스토리는 팀 협업의 핵심입니다!
